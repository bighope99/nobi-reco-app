## ワークフロー

### Step 1: 対象ページを特定する

スクリプトでページ一覧を取得して特定する（`notion-search` より確実）。

```bash
# 全マニュアルページ一覧
npx tsx .claude/skills/manual-update/scripts/query-manual-pages.ts

# キーワードで絞り込む
npx tsx .claude/skills/manual-update/scripts/query-manual-pages.ts --keyword "記録管理"
```

スクリプトが使えない場合は `notion-search` で代替:
```
query: 【記録管理】
filters: {}
```

上の一覧から該当するページを特定する。複数のページが対象になる場合はすべて更新する。

### Step 2: 既存ページに追記する

`notion-update-page` で既存ページの末尾または該当セクションに追記する。

**追記スタイルの原則**:
- 既存の内容は削除しない
- 手順が変わった場合は古い手順に取り消し線（~~旧手順~~）を入れてから新手順を追記
- 追記箇所には `（YYYY-MM-DD 更新）` の日付メモを添える
- 今日の日付: 2026-03-25

### Step 3: 対応ページがない場合は新規作成

`notion-create-pages` で新規作成する。

```json
{
  "parent": { "type": "data_source_id", "data_source_id": "260092aa-b014-8096-a3c1-000bc19e2a36" },
  "pages": [{
    "properties": {
      "名前": "【機能カテゴリ】ページタイトル",
      "タグ": "[\"マニュアル\"]",
      "関連PJ": "https://www.notion.so/2b2092aab014800d8beae48db2dae981"
    },
    "content": "（下記テンプレートを使う）"
  }]
}
```

### Step 4: Wikiインデックスページを更新する（必要な場合）

以下のどちらかに該当する場合は、Wikiインデックスページも更新する。

- **新規ページを作成した**: インデックスに新しいリンクを追加する
- **ページタイトルを変更した**: インデックスの該当リンクテキストを修正する

**Wikiインデックスページ**:
- URL: `https://www.notion.so/32e092aab014811a8b22f12e32dbc745`
- ID: `32e092aa-b014-811a-8b22-f12e32dbc745`

`notion-update-page` でインデックスを更新する例（新規リンク追加）:
```
# 追記箇所の例（施設管理者のみ → 施設設定 セクション末尾）
- [新機能名](https://www.notion.so/<新ページID>) — 一言説明
```

**インデックスの構成**（どのセクションに追加するか判断する）:
- `## はじめに` — ログイン・初期設定系
- `## 職員・施設管理者 共通の機能` — 全員が使う機能
  - `### ホーム画面` / `### 記録管理` / `### 出席管理` / `### 子ども管理`
- `## 施設管理者のみの機能` — 管理者限定の機能
  - `### 子ども管理（登録・編集）` / `### 施設設定` / `### データ管理`
- `## よくある質問` — FAQ（トグル形式）

### Step 5: スクリーンショットをユーザーに依頼

Notionへのスクリーンショット貼り付けは自動化できないため、更新後にユーザーへ伝える:

> 「〇〇のページを更新しました。画面の変更が視覚的にわかる場合は、スクリーンショットを撮って該当ページに貼り付けてください。」

## 読者向け文章のルール

対象読者はPC不慣れな施設スタッフ。以下を守る。

- **平易な言葉**: 「クリック」「選ぶ」「入力する」「確認する」を使う。「実行」「設定値」「パラメータ」などの技術用語は避ける
- **具体的に**: 「適切に設定する」ではなく「〇〇に〇〇と入力する」のように書く
- **短い文**: 1文を短くして、一度に伝える情報を減らす
- **番号付きステップ**: 手順は必ず番号付きリストで書く
- **対象ロールを明記**: ページ冒頭に「施設管理者のみ」「全員」などを書く
