# add-review-point スキル

`.claude/commands/pr-review.md` のプロジェクト固有ルール一覧に新しい考慮事項を追加する。

## 目的

CLAUDE.md はセッション毎に読み込まれコンテキストを消費する。
繰り返しチェックすべき考慮事項は `/pr-review` コマンドに集約し、レビュー時だけ参照することでコンテキストを節約する。

## 実行手順

### 1. 追加内容の確認

ユーザーから追加したい考慮事項を受け取る。不明な場合は確認する：
- **何を**チェックするか（NG パターン / OK パターン）
- **なぜ**必要か（バグ原因、CodeRabbit 指摘、プロジェクト固有事情）

### 2. pr-review.md を読む

```
.claude/commands/pr-review.md の「プロジェクト固有のルール」セクションを読む
```

### 3. 箇条書き1行で追加

`**プロジェクト固有のルール**：` のリスト末尾に追記する。

フォーマット：
```
  - {対象}: {NGパターン} → {OKパターン or 理由}
```

例：
```
  - エラーハンドリング: `console.error` のみで握り潰さないこと（ユーザーへのフィードバックが必要）
  - 日付: タイムゾーンは UTC で保存、表示時に JST 変換すること（timezone-strategy スキル参照）
```

### 4. 確認メッセージ

追加後、追加した内容を1行で報告する：
```
✅ `/pr-review` に追加しました：「{追加した内容}」
```

## 注意事項

- **1行・箇条書き**で書く（長い説明は参照スキルへのリンクで代替）
- CLAUDE.md にも同じ内容があれば、CLAUDE.md 側は削除を提案する（重複排除）
- 追加後に pr-review.md の「プロジェクト固有のルール」が10項目を超えたら、カテゴリ分けを検討する
